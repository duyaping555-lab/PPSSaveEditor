# PPSSPP Save Editor APK 构建 - 错误排查指南

## 常见错误及解决方案

### 错误1: 缺少 Android NDK

**错误信息：**
```
NDK not found or invalid
或者
ndk_dir not specified
```

**解决方案：**

方法A - 手动下载NDK：
```bash
# 使用Android Studio的SDK Manager
# 或命令行下载
```

方法B - 让buildozer自动下载：
1. 打开 `mobile_app/buildozer.spec`
2. 注释掉 `android.ndk_path` 行
3. 重新运行构建

---

### 错误2: 缺少依赖包

**错误信息：**
```
No module named 'pythonforandroid'
或者
buildozer: command not found
```

**解决方案：**
```bash
# 安装所有依赖
pip install python-for-android buildozer cython kivy pycryptodome
```

---

### 错误3: 构建超时

**错误信息：**
```
Build timed out
```

**解决方案：**
- 这是正常的，首次构建需要30-60分钟
- 保持电脑不睡眠，网络连接稳定
- 如果中断，重新运行命令会继续

---

### 错误4: 内存不足

**错误信息：**
```
OutOfMemoryError
java heap space
```

**解决方案：**
```bash
# 设置更大的内存
set GRADLE_OPTS=-Xmx4096m
# 然后重新构建
```

---

### 错误5: 网络问题

**错误信息：**
```
Connection timeout
Failed to download
```

**解决方案：**
- 检查网络连接
- 使用代理/VPN（如果需要访问国外资源）
- 重试几次

---

## 备用方案（如果所有方法都失败）

### 方案A: 使用 GitHub Actions（推荐）

1. 创建GitHub账号
2. 上传代码到GitHub
3. GitHub自动构建APK
4. 下载APK

**优点：**
- 无需本地配置
- 构建环境稳定
- 自动生成Release

### 方案B: 使用 Docker

```bash
# 安装Docker Desktop
# 然后运行：
docker run --rm -v "$(pwd)":/home/user/hostcwd kivy/buildozer:latest bash -c "cd /home/user/hostcwd/mobile_app && buildozer android debug"
```

### 方案C: 使用 Android Studio

1. 安装 Android Studio
2. 打开项目文件夹
3. 同步Gradle
4. Build > Build APK

### 方案D: 使用 Termux（手机上直接运行）

```bash
# 在Android手机上安装Termux
pkg install python
pip install pycryptodome

# 然后直接运行Python版本
python psp_save_editor.py /sdcard/PSP/SAVEDATA/XXX --decrypt --hex
```

---

## 最简单的解决方案

如果构建APK总是失败，可以直接使用 **Python 版本**：

```bash
# 在PC上
pip install pycryptodome
python psp_save_editor.py <存档路径> --decrypt --hex

# 或在手机上用Termux
```

功能完全相同，只是没有GUI界面。
